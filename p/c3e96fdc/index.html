获取 OpenAI API Key（附 one-api 部署方式）

发表于2023-12-01|更新于2023-12-10
|字数总计:1.4k|阅读时长:4分钟|阅读量:

获取 OpenAI 的 API KEY 实在是太困难了，平时想捣鼓小玩意处处受阻。但是现在国产大模型 API 已经可以很容易获取了，我再想能不能把对国产 API 的请求转接包装一下，使之成为 OpenAI API 的格式呢？

搜索一番，果然存在相关项目，而且远超于我的期望！songquanpeng/one-api(github.com) 提供以下服务以及自身亮点：

  1. 通过标准的 OpenAI API 格式访问所有的大模型
  2. 开箱即用
  3. 多渠道负载均衡
  4. 用户管理、令牌管理、兑换码管理…（简直就是做好了整个商业化的小系统）

如想立即体验一下的话，可以关注第一个部分；如果打算尝试自己部署来玩，可以直接看第二部分。

立即获取 OpenAI API Key （接口分发服务）

可前往 爱发电 购买令牌，并在 https://aigc.uuanqin.top/ 注册账户并兑换即可。

使用细节在 https://aigc.uuanqin.top/ 的首页部分或爱发电 商品页 已经有比较详细的进行说明。

当然了，目前只是测试阶段，价格设置很低，也不打算以此盈利，纯当玩了。

ECS 部署 One API 项目（Docker 方式）

在实际部署的时候，也遇到了一些困难，现分享一下部署过程。

我这里采用的是 Docker 部署方式。本文涉及到的资源不一定都是使用 one-api 必须资源，因为你可以使用其他方式部署。目前我的部署方式使用的资源包括：

  1. 拥有自己的域名（这是为了更好的访问服务）
  2. 拥有自己的服务器（硬性条件）
  3. 使用 Nginx 反向代理（主要是因为服务器用了 Nginx）

Docker

渠道增加
